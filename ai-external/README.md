スキルは、シニアエンジニアがソフトウェアを構築するときに用いるワークフロー、品質ゲート、ベストプラクティスをエンコードしたものです。ここに収めたスキル群は、AI エージェントが開発の各フェーズで一貫して同じ作法に従えるようパッケージ化してあります。

```
  定義           計画           構築           検証          レビュー        出荷
 ┌──────┐      ┌──────┐      ┌──────┐      ┌──────┐      ┌──────┐      ┌──────┐
 │案の  │ ───▶ │仕様  │ ───▶ │実装  │ ───▶ │テスト│ ───▶ │品質  │ ───▶ │本番  │
 │精錬  │      │ PRD  │      │      │      │Debug │      │Gate  │      │公開  │
 └──────┘      └──────┘      └──────┘      └──────┘      └──────┘      └──────┘
  /spec          /plan          /build        /test         /review       /ship
```

## コマンド

開発ライフサイクルに対応した 7 つの slash command を提供します。各コマンドは適切なスキルを自動的に起動します。

| やりたいこと | コマンド | 中核となる原則 |
|-------------------|---------|---------------|
| 何を作るかを定義する | `/spec` | コードより先に仕様 |
| どう作るかを計画する | `/plan` | 小さくアトミックなタスクへ |
| 段階的に作り込む | `/build` | 一度に一スライス |
| 動くことを証明する | `/test` | テストこそが証拠 |
| マージ前にレビューする | `/review` | コード健全性を高める |
| コードを単純化する | `/code-simplify` | 巧妙さより明快さ |
| 本番に届ける | `/ship` | 速いことは安全なこと |

仕様ができたあとは手作業を減らしたい場合、**`/build auto`** が計画の生成から各タスクの実装までを 1 回の承認で一気通貫に走らせます。最初に計画を承認すれば、あとは自律的に進みます。タスク**間**に人が挟まる手間がなくなるだけで、検証は省きません。各タスクは引き続きテスト駆動で進められ、個別にコミットされ、失敗や危険なステップでは自動的に一時停止します。

スキルはあなたの作業内容に応じて自動的にも起動します。API を設計していれば `api-and-interface-design` が、UI を構築していれば `frontend-ui-engineering` が、といった具合です。
