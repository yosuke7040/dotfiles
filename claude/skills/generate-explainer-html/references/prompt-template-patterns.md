# プロンプトテンプレのパターン(左ペインの中身)

iframe の **外側** はチャット UI ではなく、**コピー可能なプロンプトテンプレ** のセット。
ユーザーが *別の*(ローカルファイルを読める)AI に対して、バンドル用の **iframe ビュー
文書を 1 つ作って** と頼める。各テンプレはカードになっており、タイトル / 用途 / 本文 /
コピーボタン / 任意のタグを持つ。

`scripts/build_html.py` はこれを `prompts.json` の配列から読み、`index.html` のカードに
レンダリングする。サンプルは `sample-prompts.json`。

この skill での重要ルール:

- 各テンプレは **iframe ビュー文書を 1 つだけ** 作らせる。`index.html` 丸ごとや、シェル /
  既存プロンプトの編集はさせない。シェルを書くのは決定論的なビルドスクリプトのみ。
- テンプレは YAML を **絶対パスで参照** し、AI にそれを読ませる。**YAML 中身は埋め込まない**。
- 新しいビューは **追加型**。生成文書は `views/` に保存され、再ビルドで切り替えタブが
  既存ビューの隣に追加される。

## prompts.json のシェイプ

```json
[
  {
    "id": "view-table",
    "title": "テーブルのビューを追加",
    "description": "概念・重要度・難易度・関係・根拠を一覧で見たい人向けのビューを追加する。",
    "tags": ["view", "table"],
    "prompt": "..."
  }
]
```

- `id` — 安定 ID(プロンプト本文の DOM id にも使う)。
- `title` — カード見出し。
- `description` — このテンプレをいつ使うか。
- `prompt` — コピーされる本文。後述のプレースホルダを使える。
- `tags` — 任意のチップ。

## プレースホルダ

`build_html.py` は各 `prompt` 内の以下のトークンを **コピーされた YAML ファイルの絶対
パス** に置換する(中身ではなく)。

- `{{core_yaml_path}}` → `<bundle>/core.yaml` の絶対パス
- `{{view_yaml_path}}` → `<bundle>/view.yaml` の絶対パス

それ以外の `{{...}}` トークン(例: `{{希望する表現}}`)はそのまま残り、ユーザーが送信前に
埋める。これが「自由記述」テンプレの仕組み。

テンプレ内では次のように書いて、AI にファイルを開かせる:

```
# 参照（必ず読み込む）
- core.yaml: {{core_yaml_path}}
- view.yaml: {{view_yaml_path}}
これらのファイルを開いてから作ってください。yml の中身はこのプロンプトには貼り付けていません。
```

> 送られる側の AI は **ローカルファイル読み込みができる前提**(Claude Code / Cursor /
> エージェント IDE)。プレーン Web チャットでは設計上スコープ外。

## プロンプト本文での安全な言い回し

プロンプト本文は `index.html` に埋め込まれ、`validate_html.py` がスキャンする。
「外部依存しない」という指示は **汎用的に** 書く。本文中に禁止トークン
(`fetch(`、`XMLHttpRequest`、`localStorage`、`document.cookie`、`window.parent` /
`window.top`、`http` / `https` URL)を literal に書かない。次のように書く:

> 制約: <!DOCTYPE html> から始まる単一の自己完結 HTML document を出力。外部 CDN・外部 CSS・
> 外部スクリプト・外部読み込みの iframe を使わない。ネットワーク通信・ブラウザストレージ・
> cookie・親フレームへのアクセスを行わない。

(置換される絶対パス `/Users/...` には URL スキームが含まれないので、validator の `http(s)://`
チェックには引っかからない。)

## テーマに関する言い回し

各ビュー文書はライトデフォルトで、自分自身の URL ハッシュからテーマを読む。AI への指示:

> デフォルトはライトテーマ。URL ハッシュ `#theme=dark` を読み取ってダークにも対応する
> (自分の `location.hash` を見て、`hashchange` も listen して `data-theme` を切り替える)。

## 必須テンプレート

生成バンドルには **最低限以下のテンプレ** を入れる。各テンプレは:
`{{core_yaml_path}}` / `{{view_yaml_path}}` を読ませ、自己完結した iframe HTML を 1 つ
吐かせ(ライトデフォルト + `#theme` ハッシュ対応)、結果が新しい切替ビューとして追加される
ことを述べる。

1. **テーブル** — 概念一覧、重要度、難易度、関係、根拠、次に読むべき箇所。
2. **ワークツリー** — ツリー表示、directory / file / concept の階層、読む順番、影響の大きい
   箇所、開閉可能セクション。
3. **初心者** — 用語説明、なぜ重要か、3 ステップの導線、例え話、質問例。
4. **エンジニア** — 依存、変更ファイル、読む順番、レビュー観点、リスク、テスト。
5. **PdM / Biz** — 何が変わるか、なぜ重要か、ユーザー / 業務影響、意思決定点、リスクと確認。
6. **自由記述** — ユーザーが埋める `{{希望する表現}}` プレースホルダつき。

> 「HTML 全体を再生成する」テンプレは **意図的に置かない**。シェルとプロンプトを AI に
> 書き直させない。シェルを変えたいときは、プロンプトではなくスクリプトを変える。

## カードの表示仕様

各カード:

- タイトル
- 用途説明
- プロンプト本文(コンパクトに保つために `<details>` で折り畳み)
- コピーボタン
- 任意のタグ

コピーボタンは JS の clipboard API、フォールバックは **textarea を選択して**
`document.execCommand("copy")` する。ボタンはプロンプト本文の `textContent` を読むので、
コピーされるのは置換済みの絶対パスを含む元のテキストになる。
