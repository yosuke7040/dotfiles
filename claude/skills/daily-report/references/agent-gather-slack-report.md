# agent-gather-slack-report

`slack_read_user_profile()`（引数なし）を呼び出して、現在のユーザーの Slack ID を取得する。

## 自分のメッセージを検索

レポート対象の日付範囲に送信したメッセージを検索する:

```
slack_search_public_and_private(query="from:me", sort="timestamp", sort_dir="desc", limit=50)
```

タイムスタンプで結果をフィルタし、レポートの日付範囲に合わせる。`on:` による日付フィルタが必要な場合、クエリに日付修飾子を追加: `"from:me after:{start_date} before:{end_date}"`。

## 会話のカテゴリ分類

チャンネルごとにメッセージをグループ化し、何を伝えたかを要約する:

| 残す                      | 除外                            |
| ------------------------ | ------------------------------ |
| 実質的なディスカッション     | Bot/自動メッセージ              |
| 行われた意思決定            | 単純なリアクションや絵文字のみ    |
| 質問と回答                 | "ok"、"thanks"、"sounds good"  |
| 共有したステータス更新       | 重複するクロスポスト             |
| アナウンスやFYI投稿         | スレッドのノイズ（繰り返しフォロー）|

## チャンネルごとの要約

メッセージをチャンネルレベルの要約にグループ化する。チャンネル名をコンテキストとして使用:

| チャンネルのメッセージ                            | 要約                                  |
| ---------------------------------------------- | ------------------------------------ |
| #eng: "deployed v2", "fixed the flaky test"     | デプロイ更新とテスト修正の共有          |
| #project-x: "design looks good", "let's ship it"| デザインの承認とリリースのGO判断        |
| #team: "OOO tomorrow", "standup notes..."        | 予定の共有とスタンドアップノート        |
