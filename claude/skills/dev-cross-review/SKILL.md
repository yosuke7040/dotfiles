# PRレビュースキル

あなたはレビューオーケストレーターとして、3つのAIレビューエージェント（Claude / Codex #1 / Codex #2）を並列稼働させ、結果を統合して高品質なレビューレポートを作成する。

## 絶対ルール

- **オーケストレーターの仕事は3つのAIの結果を統合・評価すること**。エージェントが出していない情報をオーケストレーターが独自に補完しない
- **コードの変更は一切行わない**（レビューレポートの作成のみ）
- **Bashコマンドは1コマンドずつ実行する**。`cd /path && command` のように `&&` でチェーンしない。先に `cd` を単独で実行してから、次のコマンドを実行すること

## 手順

### 1. PR情報の解析とローカルリポジトリの特定

引数 `$ARGUMENTS` のURL（`https://github.com/owner/repo/pull/123` 形式）から owner/repo と PR番号を抽出する。

ローカルリポジトリのパスは `~/src/{repo}/` とする。`ls` で存在確認を行い、ディレクトリが見つからない場合はエラーメッセージを表示して終了する。

例:
- `my-org/my-app` → `~/src/my-app/`
- `another-org/api-server` → `~/src/api-server/`

### 2. PR情報の事前取得

エージェント起動前に、オーケストレーターがPR情報と差分を取得して一時ファイルに保存する。各エージェントにはこのファイルパスを渡す。

1. `gh pr view <PR番号> --repo <owner/repo> --json title,body,files` を実行し、結果を `.code-reviews/.tmp/pr-review-<PR番号>-info.json` に保存
2. `gh pr diff <PR番号> --repo <owner/repo>` を実行し、結果を `.code-reviews/.tmp/pr-review-<PR番号>-diff.patch` に保存

### 3. 3つのAIレビューエージェントを並列実行

まずオーケストレーターがステップ1で特定したローカルリポジトリに `cd` する。

次に、以下の3つを **すべて並列** で Agentツール（バックグラウンド実行、**`isolation: "worktree"`**）で起動する。各エージェントには `agents/` ディレクトリ（このスキルファイルと同じディレクトリ内）のプロンプトファイルを Read させ、その内容をレビュー指示として従わせる。オーケストレーターがレビュー指示を自分で組み立てる必要はない。

**重要**:
- 全エージェントに `isolation: "worktree"` を指定すること。これにより各エージェントがローカルリポジトリの隔離コピー（worktree）上で動作し、ローカルのブランチ状態に影響されない
- 各エージェントはプロンプトファイルの指示に従い、worktree 内で `git fetch origin pull/<PR番号>/head` → `git checkout FETCH_HEAD` を実行してPRブランチに切り替える（detached HEAD。`gh pr checkout` だとブランチ名が worktree 間で競合するため）。これにより周辺コード参照時の偽陽性を防ぐ
- worktree の作成・削除は Agent ツールが自動管理する

---

#### Agent 1: Claude（Agentツール、バックグラウンド実行、`isolation: "worktree"`）

Agentツールでバックグラウンド起動する。プロンプト:

```
以下のファイルを Read ツールで読み込み、その内容をレビュー指示として従ってください: <agents/claude-reviewer.md の絶対パス>

PR番号: <PR番号>
リポジトリ: <owner/repo>
PR情報: <PR情報ファイルの絶対パス>
差分: <差分ファイルの絶対パス>
```

---

#### Agent 2: Codex #1（Agentツール、バックグラウンド実行、`isolation: "worktree"`）

エージェント内で `codex exec` が実行されるため、Codex の冗長な中間ログ（thinking, exec等）がメインコンテキストに入らず、最終レビュー結果のみが返る。

Agentツールでバックグラウンド起動する。プロンプト:

```
以下のファイルを Read ツールで読み込み、その内容をレビュー指示として従ってください: <agents/codex-reviewer.md の絶対パス>

PR番号: <PR番号>
リポジトリ: <owner/repo>
PR情報: <PR情報ファイルの絶対パス>
差分: <差分ファイルの絶対パス>
```

---

#### Agent 3: Codex #2（Agentツール、バックグラウンド実行、`isolation: "worktree"`）

Agent 2 と同一のプロンプトで、別のAgentとしてバックグラウンド起動する。同じモデルでも非決定性により異なる観点の指摘が得られることを期待する。

Agentツールでバックグラウンド起動する。プロンプト:

```
以下のファイルを Read ツールで読み込み、その内容をレビュー指示として従ってください: <agents/codex-reviewer.md の絶対パス>

PR番号: <PR番号>
リポジトリ: <owner/repo>
PR情報: <PR情報ファイルの絶対パス>
差分: <差分ファイルの絶対パス>
```

### 4. レビュー結果の統合（オーケストレーション）

3つのAIの結果を収集し、**オーケストレーターとして自分自身の判断を加えて** 統合する。

> **クロスレビューの目的**: 複数のAIレビューエージェントを並列で走らせることで、1つのAIだけでは見落としていた問題を拾う。

#### 4-1. 各AIの指摘を一覧化する

各AIの指摘を以下の情報で整理する:
- 指摘元（Claude / Codex #1 / Codex #2）
- 指摘箇所（ファイル名・該当コード）
- 指摘内容

#### 4-2. クロスリファレンス

- **複数のAIが同じ箇所を指摘** → 信頼度が高い
- **1つのAIだけが指摘** → その事実をレポートに明記する（検出元が1つである旨を付記）
- **AIの指摘が矛盾** → 矛盾している事実と各AIの主張をそのままレポートに記載し、判断は人間に委ねる

#### 4-3. 深刻度の判定

| レベル | 意味 | 基準 |
|--------|------|------|
| CRITICAL | バグ | データ不整合、意図と異なる動作を引き起こすコードが含まれている |
| WARNING | 潜在的な問題 | コード自体は正しく動くが、特定条件や将来の変更で壊れやすい設計/実装になっている |
| INFO | 改善の余地 | パフォーマンス、冗長コード、テスト不足 |

> **判定ルール**: CRITICALは「そのコードが現在実行されるか」ではなく「PRの差分にバグとなるコードが含まれているか」で判定する。TODO・コメントアウト等で現在無効でも、コード自体が誤動作を起こすならCRITICAL。

### 5. 出力

レビュー結果を**ファイル**と**ターミナル**の両方に出力する。

#### 出力先

```
.code-reviews/{YYYYMMDD_HHMMSS}_{repo}_PR{番号}.md
```

- `{YYYYMMDD_HHMMSS}`: スキル実行時の日時をJSTで取得する（`TZ=Asia/Tokyo date '+%Y%m%d_%H%M%S'`）

#### ファイルの内容

```markdown
# PRレビュー: {PRタイトル}

- PR: {PR URL}
- レビュー日時: {YYYY-MM-DD HH:MM:SS}
- レビュアー: Claude, Codex #1, Codex #2

## サマリー

| 深刻度 | 件数 |
|--------|------|
| CRITICAL | {N} |
| WARNING | {N} |
| INFO | {N} |

## 指摘事項

### [{深刻度}] {指摘タイトル}

**検出元**: {Claude / Codex #1 / Codex #2 のどれが検出したか。複数の場合はすべて記載}

**該当箇所**: `{ファイルパス}` — {該当コードの説明}

**問題**: {何が問題か}

**根拠**: {なぜ問題と判断したか。該当コードを引用して説明}
```

### 6. 後処理

ステップ2で作成した一時ファイルを削除する:
- `.code-reviews/.tmp/pr-review-<PR番号>-info.json`
- `.code-reviews/.tmp/pr-review-<PR番号>-diff.patch`

## 補足

- このスキルはあくまで自動レビューであり、人間のレビューを代替するものではない
- 偽陽性よりも偽陰性を避けることを優先する。怪しいものは指摘し、判断は人間に委ねる
