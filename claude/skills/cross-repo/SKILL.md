# Cross-Repo: 兄弟リポジトリ参照スキル

現在のリポジトリのナレッジ（CLAUDE.md、設計文書、コード等）をフル活用しつつ、兄弟リポジトリのコードやドキュメントを参照して作業する。

**核心原則: ナレッジが一番濃いリポジトリで作業する。兄弟リポジトリは参照のみ。**

---

## 制約

- **兄弟リポジトリは読み取り専用**。Write / Edit / Bash による変更は絶対に行わない
- **編集は現在のリポジトリのみ**
- 兄弟リポジトリの `.env`、secrets、credentials は読まない

---

## 手順

### Step 1: 兄弟リポジトリの検証

引数 `$ARGUMENTS` からパスを取得し、以下を確認する:

```bash
# 存在確認
ls $ARGUMENTS

# gitリポジトリかどうか
git -C $ARGUMENTS rev-parse --is-inside-work-tree
```

パスが存在しないかgitリポジトリでない場合、エラーメッセージを表示して終了。

### Step 2: 兄弟リポジトリの全体把握

ファイル構造を取得してリポジトリの全体像を把握する:

```bash
rg --files $ARGUMENTS --sort path | head -200
```

### Step 3: ナレッジの読み込み

兄弟リポジトリの以下のファイルが存在すれば読み込む（優先順）:

1. `CLAUDE.md`（およびサブディレクトリの `CLAUDE.md`）
2. `README.md`
3. `package.json` / `Cargo.toml` / `go.mod` / `pyproject.toml` 等のプロジェクト設定
4. `.claude/settings.json`

これにより兄弟リポジトリの技術スタック、規約、アーキテクチャを理解する。

### Step 4: コンテキストの統合とユーザーへの報告

以下を簡潔に報告する:

- 兄弟リポジトリ名と概要
- 検出した技術スタック
- 主要なディレクトリ構成
- 現在のリポジトリとの関連性（共通の依存、API境界など）

報告後、ユーザーの指示を待つ。

### Step 5: 作業の実行

ユーザーの指示に従い、**両方のリポジトリのナレッジを活用して現在のリポジトリで作業する**。

兄弟リポジトリのコードを参照する際は:
- `Read` で直接ファイルを読む
- `Grep` / `Glob` でコードを検索する
- `rg` でパターンを横断検索する

例:
- 兄弟リポジトリのAPI定義を見て、こちら側のクライアントコードを書く
- 兄弟リポジトリの設計文書を参照して、こちら側の実装方針を決める
- OSSリポジトリの内部実装を読んで、こちら側で適切に利用する

---

## 使用例

```
# 兄弟のAPIサーバーリポジトリを参照しながらフロントエンドを開発
/cross-repo ../api-server

# OSSライブラリの内部実装を参照しながら利用コードを書く
/cross-repo ../open-source-lib

# 戦略文書・デザイン文書リポジトリを参照しながらコーディング
/cross-repo ../docs-private
```
