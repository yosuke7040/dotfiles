code-simplification スキルを起動する。

直近で変更されたコード（または指定された範囲）を、正確な振る舞いを保ったまま単純化する:

1. CLAUDE.md または AGENTS.md を読み、プロジェクトの慣習を把握する
2. 対象コードを特定する。広い範囲が指定されていない限り、直近の変更を対象にする
3. 触る前に、コードの目的、呼び出し元、エッジケース、テストカバレッジを理解する
4. 単純化の機会を探す:
   - 深いネスト → ガード節または抽出したヘルパー
   - 長い関数 → 責務ごとに分割
   - ネストした三項演算子 → if/else または switch
   - 汎用的な名前 → 説明的な名前
   - 重複したロジック → 共有関数
   - デッドコード → 確認後に削除
5. 単純化は一つずつ適用する。各変更後にテストを実行する
6. すべてのテストが通り、ビルドが成功し、差分がきれいであることを検証する

単純化後にテストが失敗した場合は、その変更を戻して再検討する。結果のレビューには `code-review-and-quality` を使う。
